# ZEVM

ZEVM is a docs-first Zig Ethereum client with two runtime modes:

- trusted mode for the phase-1 local dev node, local mutation, and optional fork-backed development
- light mode for the phase-2 consensus-backed verified-read client

Forking is configuration inside trusted mode, not a third product mode.

## What These Docs Promise

These docs define the intended ZEVM contract, publishing exact public behavior where the repo authority settles it and explicit blocking questions where it does not, including:

- exact startup flags and config-file fields
- defaults and precedence rules
- invalid combinations and startup failure behavior
- HTTP JSON-RPC transport and notification behavior
- mode-specific method availability and block-tag semantics
- canonical `zevm_*` trusted-mode controls, with compatibility aliases captured in the repo-local reference pages
- explicit phase-3 deferral for advanced tooling that is not part of the phase-1 contract
- exact separation between intended behavior and current `HEAD`

## Repo Authority

The repo-local authority order is PRD first, `docs-first-process` second, current `src/` and tests third as contradiction evidence, and older issue/context/plan docs last as non-normative evidence. Maintainer decisions and other control docs backfill exact detail or record explicit contradiction plus public-doc stance for a surface, but they do not silently override that hierarchy. The public docs here are the contract surface; they do not depend on an external spec file for their exact startup and configuration claims.
