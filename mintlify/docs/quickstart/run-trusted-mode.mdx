# Run Trusted Mode

Trusted mode is the default ZEVM runtime. The source-build contract installs `zevm` to `./zig-out/bin/zevm`.

## Intended Behavior

Trusted mode is the writable local dev-node runtime. Its canonical nonstandard methods are `zevm_*`, and the accepted compatibility aliases are documented in the repo-local reference pages.

### Intended Startup

Trusted mode starts by default:

```bash
./zig-out/bin/zevm
```

The equivalent explicit startup is:

```bash
./zig-out/bin/zevm --mode trusted --host 127.0.0.1 --port 8545
```

If a config file already selects `mode.trusted`, `--mode trusted` is optional and only confirms the branch.

Trusted mode also accepts the phase-1 development flags defined by the PRD:

```bash
./zig-out/bin/zevm --mode trusted --mining manual
```

```bash
./zig-out/bin/zevm --mode trusted --mining interval --block-time 12
```

```bash
./zig-out/bin/zevm --mode trusted --fork-url https://rpc.example --fork-block-number 22000000
```

### Mining Modes

`--mining` maps to `mode.trusted.mining`:

- `auto` uses automatic block production
- `manual` mines only when requested
- `interval` requires `--block-time <seconds>`

`--block-time` is only valid with `--mining interval`.
`--mining interval` without `--block-time` fails startup.

### Forked Startup

Forking stays inside trusted mode and does not change `chainId`.

- `--fork-url` mirrors `mode.trusted.fork.url`
- `--fork-block-number` mirrors `mode.trusted.fork.blockNumber`
- omitting the block number forks from the upstream latest block
- `--fork-block-number` without `--fork-url` is a hard startup failure

### Defaults

| Setting | Default |
| --- | --- |
| `chainId` | `31337` |
| `coinbaseIndex` | `0` |
| `initialBalance` | `10000000000000000000000` |
| `gasPrice` | `2000000000` |
| `baseFee` | `1000000000` |
| `blobBaseFee` | `1` |
| `maxPriorityFeePerGas` | `1000000000` |
| `blockGasLimit` | `30000000` |
| `mining` | `auto` |

`coinbaseIndex` defaults to `0` and valid values are `0` through `9`.

### Managed Accounts

The exact 10-account managed-wallet table is part of the trusted-mode contract. `coinbaseIndex` selects from that table, `eth_accounts` returns it in ascending index order, and `eth_sendTransaction` signs only for those managed accounts unless impersonation is active.

### First Successful Calls

These are contract examples for the first calls that should succeed against a fresh trusted node once the documented startup path exists. They are not a claim about current runnable `HEAD`.

The quickstart-owned first-call surface is the trusted identity and default-read baseline: `eth_chainId`, `eth_blockNumber`, `eth_accounts`, `eth_coinbase`, `eth_getBalance`, `eth_gasPrice`, `eth_maxPriorityFeePerGas`, and `eth_blobBaseFee`.

```json
{ "jsonrpc": "2.0", "id": 1, "method": "eth_chainId", "params": [] }
```

Returns `0x7a69` by default.

```json
{ "jsonrpc": "2.0", "id": 2, "method": "eth_blockNumber", "params": [] }
```

Returns `0x0` on a fresh trusted node before additional mining.

```json
{ "jsonrpc": "2.0", "id": 3, "method": "eth_accounts", "params": [] }
```

Returns the exact 10-address managed-account table above in ascending index order.

```json
{ "jsonrpc": "2.0", "id": 4, "method": "eth_coinbase", "params": [] }
```

Returns the managed account selected by `coinbaseIndex`, which is account `0` by default.

```json
{ "jsonrpc": "2.0", "id": 5, "method": "eth_getBalance", "params": ["0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266", "latest"] }
```

Returns that account's configured trusted-mode balance at the current local head. Exact tuple and error rules for these calls live on [Core Reads](../reference/json-rpc/core-reads).

```json
{ "jsonrpc": "2.0", "id": 6, "method": "eth_gasPrice", "params": [] }
```

Returns `0x77359400` by default. On the same fresh node, `eth_maxPriorityFeePerGas` returns `0x3b9aca00` and `eth_blobBaseFee` returns `0x1`.

### Validation Rules

Startup fails before opening the listener when:

- `--block-time` is supplied without `--mining interval`
- `--fork-block-number` is supplied without `--fork-url`
- `coinbaseIndex` is outside `0..9`
- any trusted-only field is supplied in light mode
