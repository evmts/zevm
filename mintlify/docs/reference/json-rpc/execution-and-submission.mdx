# Execution And Submission

<CurrentHeadNote />

## Methods

| Method or surface | Contract |
| --- | --- |
| `eth_call` | simulate execution without mutating canonical trusted state |
| `eth_estimateGas` | estimate gas against trusted state without mutating canonical state |
| `eth_sendTransaction` | submit a transaction from a managed account or active impersonation |
| `eth_sendRawTransaction` | submit a pre-signed transaction |
| automine | mine immediately after each accepted ready transaction |
| manual mining | accept transactions without mining automatically |
| interval mining | mine on the configured block-time interval |

## Simulation Rules

- `eth_call` and `eth_estimateGas` run on a checkpoint-and-revert path
- state overrides are part of the contract
- simulation must not mutate canonical local state

## Submission Rules

- `eth_sendTransaction` uses ZEVM signing authority only for the canonical 10-account managed wallet or active impersonation
- `eth_sendRawTransaction` accepts user-supplied signatures
- pending ordering is nonce-aware
- accepted transactions enter the trusted pending set before mining consumes them
