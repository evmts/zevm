# Deferred Surfaces

<CurrentHeadNote />

## Deferred Scope

These surfaces remain intentionally deferred until trusted-mode fundamentals and light-mode verified reads are stable:

- `debug_traceCall`
- `debug_traceTransaction`
- filter lifecycle APIs such as `eth_newFilter` and `eth_getFilterChanges`
- subscription APIs such as `eth_subscribe` and `eth_unsubscribe`
- WebSocket transport

## Why They Are Deferred

ZEVM prioritizes:

- exact startup and configuration behavior
- one canonical HTTP JSON-RPC transport
- trusted-mode execution correctness
- light-mode verified reads and status behavior

Deferred does not mean forgotten. It means these surfaces are real scope, but not phase-1 or phase-2 shipping scope.
