# Core Reads

Phase 1 trusted mode exposes exact read-only JSON-RPC methods against local state. This page is self-contained for client integrators: the method tuples, selector rules, return payloads, and error rules are spelled out here.

This page defines trusted-mode semantics for these method names. `eth_chainId`, `eth_blockNumber`, `eth_getBalance`, `eth_getCode`, `eth_getStorageAt`, and `eth_getTransactionCount` also exist in light mode with the separate contract documented on [Verified Light-Mode Reads](./verified-light-mode-reads).

## Trusted-Mode Selectors

| Selector | Meaning in trusted mode |
| --- | --- |
| `latest` | current canonical local head |
| `pending` | compatibility alias of `latest` |
| `safe` | compatibility alias of `latest` |
| `finalized` | compatibility alias of `latest` |
| `earliest` | block `0` |
| numeric quantity | exact local block number |

Malformed selectors fail with `-32602`.

## Exact Type Rules

- `QuantityHex` is a `0x`-prefixed unsigned integer with no leading zero padding except `0x0`.
- `Address` is a `0x`-prefixed 20-byte hex address.
- `Bytes32` is a `0x`-prefixed 32-byte hex value.
- `HexData` is a `0x`-prefixed hex byte string, including `0x` for empty bytes.

## Exact Method Surface

| Method | Exact params | Exact result | Exact error rules |
| --- | --- | --- | --- |
| `eth_chainId` | `[]` or omitted | `QuantityHex` | non-empty params fail with `-32602` |
| `eth_blockNumber` | `[]` or omitted | `QuantityHex` | non-empty params fail with `-32602` |
| `eth_accounts` | `[]` or omitted | array of the 10 managed trusted-mode addresses in ascending index order | non-empty params fail with `-32602` |
| `eth_coinbase` | `[]` or omitted | `Address` | non-empty params fail with `-32602` |
| `eth_gasPrice` | `[]` or omitted | `QuantityHex` | non-empty params fail with `-32602` |
| `eth_maxPriorityFeePerGas` | `[]` or omitted | `QuantityHex` | non-empty params fail with `-32602` |
| `eth_blobBaseFee` | `[]` or omitted | `QuantityHex` | non-empty params fail with `-32602` |
| `eth_getBalance` | `[address, block]` | `QuantityHex` | malformed address or selector fail with `-32602` |
| `eth_getCode` | `[address, block]` | `HexData` | malformed address or selector fail with `-32602` |
| `eth_getStorageAt` | `[address, slot, block]` | `Bytes32` | malformed address, slot, or selector fail with `-32602` |
| `eth_getTransactionCount` | `[address, block]` | `QuantityHex` | malformed address or selector fail with `-32602` |
| `eth_feeHistory` | `[blockCount, newestBlock]` or `[blockCount, newestBlock, rewardPercentiles]` where `blockCount` is `QuantityHex`, `newestBlock` uses the trusted selector set, and `rewardPercentiles` is an optional JSON array of numbers | `FeeHistoryResult` | malformed count, selector, or percentiles fail with `-32602` |

`eth_getBalance`, `eth_getCode`, `eth_getStorageAt`, `eth_getTransactionCount`, and `eth_feeHistory` use the trusted-mode selector set above. `pending`, `safe`, and `finalized` are compatibility aliases of `latest`; they do not imply consensus finality.

## Mode Scope

- `eth_accounts`, `eth_coinbase`, `eth_gasPrice`, `eth_maxPriorityFeePerGas`, `eth_blobBaseFee`, and `eth_feeHistory` are trusted-mode only and fail with `-32010` in light mode.
- `eth_chainId`, `eth_blockNumber`, `eth_getBalance`, `eth_getCode`, `eth_getStorageAt`, and `eth_getTransactionCount` use the trusted-mode semantics on this page in trusted mode and use the separate light-mode contract on [Verified Light-Mode Reads](./verified-light-mode-reads) in light mode.
- In trusted mode, malformed selectors and malformed request-shape details fail with the method-specific `-32602` rules above.

## Exact Return Shapes

### `FeeHistoryResult`

```json
{
  "oldestBlock": "0x0",
  "baseFeePerGas": ["0x3b9aca00", "0x3b9aca00"],
  "gasUsedRatio": [0.0],
  "reward": [["0x0"]]
}
```

Field rules:

- `oldestBlock` is `QuantityHex`
- `baseFeePerGas` is an array of `QuantityHex` values with length `blockCount + 1`
- `gasUsedRatio` is an array of JSON numbers with length `blockCount`
- `reward`, when present, is an array with one item per returned block, and each item is an array of `QuantityHex` values whose length matches `rewardPercentiles`
