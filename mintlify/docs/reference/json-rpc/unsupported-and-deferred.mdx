# Unsupported And Deferred

This page tracks surfaces that remain intentionally out of scope for the trusted-mode and light-mode contracts already defined elsewhere. `eth_getLogs` is not part of this deferred set; it is supported in trusted mode and documented on the blocks, receipts, and logs page.

## Intended Contract

The deferred set includes:

- debug tracing: `debug_traceCall`, `debug_traceTransaction`
- filter lifecycle APIs beyond `eth_getLogs`: `eth_newFilter`, `eth_getFilterChanges`, `eth_getFilterLogs`, `eth_uninstallFilter`, `eth_newBlockFilter`, `eth_newPendingTransactionFilter`
- subscriptions: `eth_subscribe`, `eth_unsubscribe`
- WebSocket transport
- advanced helpers: `zevm_enableTraces`, `zevm_addCompilationResult`, `zevm_setPrevRandao`
- accepted deferred aliases for those advanced helpers: `anvil_enableTraces`, `hardhat_addCompilationResult`, `hardhat_setLoggingEnabled`, and `hardhat_setPrevRandao`

`hardhat_setLoggingEnabled` is treated as the deferred compatibility alias for `zevm_enableTraces`.

This page intentionally does not define request tuples, return payloads, or error details for those deferred surfaces.

Deferred does not mean forgotten. It means the surface exists in the product plan, but it is not part of the first shipping trusted-mode or light-mode contracts.
