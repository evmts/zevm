# Trusted-Mode Configuration

## Intended Behavior

Trusted mode is ZEVM's phase-1 writable local dev-node runtime. Its canonical nonstandard methods are `zevm_*`, and the accepted compatibility-alias inventory is documented in the repo-local reference pages.

Trusted-mode config lives under `mode.trusted`.

If a config file already selects `mode.trusted`, `--mode trusted` is optional and may confirm that branch, but it may not conflict with it.

If `--config` points to a missing file, an unreadable file, or malformed JSON, ZEVM fails startup before opening the listener, exits non-zero, reports an operator-facing error that names the path and failure class, and does not fall back to defaults.

## Field Map

| CLI flag | Config field | Default | Notes |
| --- | --- | --- | --- |
| `--chain-id` | `mode.trusted.chainId` | `31337` | top-level identity for the local chain |
| `--coinbase-index` | `mode.trusted.coinbaseIndex` | `0` | selects a managed account; default is `0` and valid values are `0..9` |
| `--initial-balance` | `mode.trusted.initialBalance` | `10000000000000000000000` | wei string |
| `--gas-price` | `mode.trusted.gasPrice` | `2000000000` | wei string |
| `--base-fee` | `mode.trusted.baseFee` | `1000000000` | wei string |
| `--blob-base-fee` | `mode.trusted.blobBaseFee` | `1` | wei string |
| `--max-priority-fee-per-gas` | `mode.trusted.maxPriorityFeePerGas` | `1000000000` | wei string |
| `--block-gas-limit` | `mode.trusted.blockGasLimit` | `30000000` | integer |
| `--mining` | `mode.trusted.mining.type` | `auto` | `auto`, `manual`, or `interval`; `interval` requires `blockTime` |
| `--block-time` | `mode.trusted.mining.blockTime` | none | seconds, only valid with `interval` |
| `--fork-url` | `mode.trusted.fork.url` | none | upstream RPC endpoint |
| `--fork-block-number` | `mode.trusted.fork.blockNumber` | none | optional fork pin; hard-fails without `--fork-url` |

Trusted-mode literal domains are exact:

- `chainId`, `coinbaseIndex`, `blockGasLimit`, `blockTime`, and `fork.blockNumber` are integer fields
- `initialBalance`, `gasPrice`, `baseFee`, `blobBaseFee`, and `maxPriorityFeePerGas` are wei decimal strings
- malformed or out-of-range integer and wei literals are hard startup failures under the shared startup contract

## Mining Shapes

`mode.trusted.mining` accepts only these JSON shapes:

```json
{ "type": "auto" }
```

```json
{ "type": "manual" }
```

```json
{ "type": "interval", "blockTime": 12 }
```

`blockTime` is an integer number of seconds and is valid only when `type` is `interval`.

## Fork Shapes

`mode.trusted.fork` accepts only these JSON shapes:

```json
null
```

```json
{ "url": "https://rpc.example" }
```

```json
{ "url": "https://rpc.example", "blockNumber": 22000000 }
```

`fork.url` mirrors CLI `--fork-url`. `fork.blockNumber` mirrors CLI `--fork-block-number`. Omitting `blockNumber` means fork from the upstream latest block. `fork` does not implicitly change `chainId`.

## Validation Rules

Startup fails when:

- an unknown CLI flag is supplied
- `chainId`, `coinbaseIndex`, `blockGasLimit`, `blockTime`, or `fork.blockNumber` uses an invalid or out-of-range integer literal
- `initialBalance`, `gasPrice`, `baseFee`, `blobBaseFee`, or `maxPriorityFeePerGas` uses an invalid or out-of-range wei literal
- `blockTime` is supplied outside `interval` mining
- `interval` mining is selected without `blockTime`
- `--block-time` is supplied without `--mining interval`
- `fork.blockNumber` is supplied without `fork.url`
- `coinbaseIndex` is outside `0..9`
- any trusted-only field is supplied in light mode

The fork hard-failure rule applies to both CLI input and config-file input.
