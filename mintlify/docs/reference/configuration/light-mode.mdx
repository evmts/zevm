# Light-Mode Configuration

## Intended Behavior

Default startup remains trusted mode; `mode.light` is explicit.

CLI `--mode light` can itself select light mode. If a config file already selects `mode.light`, `--mode light` is optional and may confirm that branch, but it may not conflict with it.

If `--config` points to a missing file, an unreadable file, or malformed JSON, ZEVM fails startup before opening the listener, exits non-zero, reports an operator-facing error that names the path and failure class, and does not fall back to defaults.

### Exact Fields

| CLI flag | Config field | Default |
| --- | --- | --- |
| `--network` | `mode.light.network` | `mainnet` |
| `--consensus-rpc-url` | `mode.light.consensusRpcUrl` | required |
| `--checkpoint` | `mode.light.checkpoint` | none |
| `--checkpoint-dir` | `mode.light.checkpointDir` | `.zevm/checkpoints/<network>` |
| `--max-checkpoint-age-seconds` | `mode.light.maxCheckpointAgeSeconds` | `1209600` |
| `--strict-checkpoint-age` | `mode.light.strictCheckpointAge` | `false` |

`--network` and `mode.light.network` accept only `mainnet`, `sepolia`, or `holesky`. Any other value is invalid and fails startup before ZEVM listens.

`maxCheckpointAgeSeconds` must be a valid `u64`. Malformed or out-of-range integer literals fail startup before ZEVM listens.

`strictCheckpointAge` is a boolean. `checkpoint` must be a `0x`-prefixed 32-byte hex string when provided.

### Allowed Network Values

| Value | Chain ID | Default checkpoint dir |
| --- | --- | --- |
| `mainnet` | `1` | `.zevm/checkpoints/mainnet` |
| `sepolia` | `11155111` | `.zevm/checkpoints/sepolia` |
| `holesky` | `17000` | `.zevm/checkpoints/holesky` |

### Checkpoint Precedence

Light mode selects its checkpoint in this order:

1. explicit user checkpoint from CLI or config
2. persisted checkpoint from `checkpointDir/checkpoint`
3. baked network default

### Persisted Checkpoint Contract

The persisted checkpoint file is part of the startup contract:

- path: `checkpointDir/checkpoint`
- contents: exactly 64 hex characters
- `0x` prefix: not stored on disk
- surrounding whitespace is ignored when loading
- malformed persisted data is a startup failure

### Baked Network Defaults

ZEVM ships baked default checkpoints for `mainnet`, `sepolia`, and `holesky`, and those defaults participate in precedence after explicit and persisted checkpoints.

The public contract does not freeze the exact literal default hashes. Current code may update those implementation defaults in `src/consensus_sync.zig` without changing the public compatibility contract, as long as baked defaults still exist and obey the documented precedence rules.

### Validation Rules

ZEVM fails startup when:

- an unknown CLI flag is supplied
- `network` is outside `mainnet`, `sepolia`, or `holesky`
- `consensusRpcUrl` is missing in light mode
- `maxCheckpointAgeSeconds` uses an invalid or out-of-range integer literal
- a trusted-mode-only field is supplied
- `checkpoint` is not a `0x`-prefixed 32-byte hex string
- the persisted checkpoint file is malformed
- the selected checkpoint age is greater than `maxCheckpointAgeSeconds` and `strictCheckpointAge = true`

When `strictCheckpointAge = false`, a stale checkpoint emits a warning and startup continues. A checkpoint whose age is exactly `maxCheckpointAgeSeconds` is still valid by contract, even though the current helper code still treats equality as stale.

### Example

```json
{
  "rpc": {
    "host": "127.0.0.1",
    "port": 8545
  },
  "mode": {
    "light": {
      "network": "sepolia",
      "consensusRpcUrl": "https://beacon.example",
      "checkpoint": null,
      "checkpointDir": ".zevm/checkpoints/sepolia",
      "maxCheckpointAgeSeconds": 1209600,
      "strictCheckpointAge": false
    }
  }
}
```
