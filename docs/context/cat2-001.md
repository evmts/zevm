# Context: cat2-001 — Fix voltaire eth_accounts Result type

## Ticket Info
- **Ticket ID**: cat2-001
- **Category**: spec-compliance
- **Goal**: Fix `eth_accounts` Result type in voltaire from `types.Quantity` (hex integer) to `[]types.Address` (array of 20-byte addresses).

---

## The Bug

**File**: `../voltaire/packages/voltaire-zig/src/jsonrpc/eth/accounts/eth_accounts.zig`

Current (wrong):
```zig
pub const Result = struct {
    value: types.Quantity,   // ← WRONG: Quantity is a hex-encoded integer

    pub fn jsonStringify(self: Result, jws: *std.json.Stringify) !void {
        try jws.write(self.value);
    }

    pub fn jsonParseFromValue(allocator: std.mem.Allocator, source: std.json.Value, options: std.json.ParseOptions) !Result {
        return Result{
            .value = try std.json.innerParseFromValue(types.Quantity, allocator, source, options),
        };
    }
};
```

Required (correct): Result must hold `[]types.Address` — an array of hex-encoded 20-byte addresses.

## Pattern: Single Address Result (correct reference)

`eth_coinbase` correctly returns a single `types.Address`:
```zig
// ../voltaire/packages/voltaire-zig/src/jsonrpc/eth/coinbase/eth_coinbase.zig
pub const Result = struct {
    value: types.Address,

    pub fn jsonStringify(self: Result, jws: *std.json.Stringify) !void {
        try jws.write(self.value);
    }

    pub fn jsonParseFromValue(allocator: std.mem.Allocator, source: std.json.Value, options: std.json.ParseOptions) !Result {
        return Result{
            .value = try std.json.innerParseFromValue(types.Address, allocator, source, options),
        };
    }
};
```

`eth_accounts` is like `eth_coinbase` but returns an **array** of addresses.

---

## Required Change

**File to edit**: `../voltaire/packages/voltaire-zig/src/jsonrpc/eth/accounts/eth_accounts.zig`

Change `Result` to hold `[]types.Address` (allocated slice):

```zig
pub const Result = struct {
    /// Array of addresses owned by the client
    value: []types.Address,

    pub fn jsonStringify(self: Result, jws: *std.json.Stringify) !void {
        try jws.beginArray();
        for (self.value) |addr| {
            try jws.write(addr);
        }
        try jws.endArray();
    }

    pub fn jsonParseFromValue(allocator: std.mem.Allocator, source: std.json.Value, options: std.json.ParseOptions) !Result {
        if (source != .array) return error.UnexpectedToken;
        const items = source.array.items;
        const addrs = try allocator.alloc(types.Address, items.len);
        for (items, 0..) |item, i| {
            addrs[i] = try std.json.innerParseFromValue(types.Address, allocator, item, options);
        }
        return Result{ .value = addrs };
    }
};
```

**Note**: The slice is heap-allocated via the provided allocator. Callers are responsible for freeing `result.value` when done.

## Zig Style Notes
- Do NOT alias `const Address = types.Address`. Use `types.Address` inline throughout.
- Do NOT store the allocator. It is passed to `jsonParseFromValue` only.
- The slice `[]types.Address` is heap-allocated; callers must free it.
