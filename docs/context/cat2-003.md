# Context: cat2-003 — Add NodeConfig struct to zevm

## Ticket Info
- **Ticket ID**: cat2-003
- **Category**: architecture
- **Goal**: Create a `NodeConfig` struct in zevm that holds node-level configuration data required for `eth_chainId`, `eth_gasPrice`, `eth_coinbase`, and `eth_accounts` RPC methods.

---

## Required Fields

Based on the ticket description and reference implementations, the `NodeConfig` struct needs:

| Field | Type | Description |
|-------|------|-------------|
| `chain_id` | `u64` | The chain ID of the current network |
| `coinbase` | `primitives.Address` | The client coinbase address (fee recipient) |
| `base_fee_per_gas` | `u256` | The current base fee per gas in wei |
| `managed_accounts` | `[]primitives.Address` | Array of addresses owned by the client |

## Proposed NodeConfig Structure

**File**: `src/node_config.zig`

```zig
const std = @import("std");
const primitives = @import("voltaire").primitives;

/// Node-level configuration for ZEVM dev node.
/// Holds values returned by eth_chainId, eth_gasPrice, eth_coinbase, eth_accounts.
pub const NodeConfig = struct {
    /// The chain ID of the current network (returned by eth_chainId)
    chain_id: u64,
    
    /// The client coinbase address (returned by eth_coinbase)
    coinbase: primitives.Address,
    
    /// The current base fee per gas in wei (returned by eth_gasPrice in legacy mode)
    base_fee_per_gas: u256,
    
    /// Addresses owned by the client (returned by eth_accounts)
    /// This slice is owned by the NodeConfig and should be freed when done
    managed_accounts: []primitives.Address,
    
    /// Returns the chain ID as a JSON-RPC Quantity
    pub fn getChainIdQuantity(self: NodeConfig) std.json.Value {
        // Format as hex quantity "0x..."
        var buf: [50]u8 = undefined;
        const hex = std.fmt.bufPrint(&buf, "0x{x}", .{self.chain_id}) catch unreachable;
        return std.json.Value{ .string = hex };
    }
    
    /// Returns the gas price as a JSON-RPC Quantity
    pub fn getGasPriceQuantity(self: NodeConfig) std.json.Value {
        // Format as hex quantity "0x..."
        var buf: [66]u8 = undefined;
        const hex = std.fmt.bufPrint(&buf, "0x{x}", .{self.base_fee_per_gas}) catch unreachable;
        return std.json.Value{ .string = hex };
    }
    
    /// Returns a slice of managed accounts
    pub fn getManagedAccounts(self: NodeConfig) []primitives.Address {
        return self.managed_accounts;
    }
};

/// Default configuration for local dev node (Hardhat/Anvil compatible)
pub const DEFAULT_CHAIN_ID: u64 = 31337;
pub const DEFAULT_BASE_FEE_PER_GAS: u256 = 1_000_000_000; // 1 gwei

/// Hardhat's first default account address
pub const DEFAULT_COINBASE: primitives.Address = primitives.Address.fromHex(
    "0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266"
) catch unreachable;
```

## Integration with RPC Handlers

The `NodeConfig` will be used by the RPC handler implementations:

```zig
// Example usage in an RPC handler
pub fn handleChainId(config: NodeConfig) !eth_chainId.Result {
    return eth_chainId.Result{
        .value = .{ .value = config.getChainIdQuantity() },
    };
}

pub fn handleGasPrice(config: NodeConfig) !eth_gasPrice.Result {
    return eth_gasPrice.Result{
        .value = .{ .value = config.getGasPriceQuantity() },
    };
}

pub fn handleCoinbase(config: NodeConfig) !eth_coinbase.Result {
    return eth_coinbase.Result{
        .value = config.coinbase,
    };
}

pub fn handleAccounts(config: NodeConfig, allocator: std.mem.Allocator) !eth_accounts.Result {
    // Convert primitives.Address to jsonrpc types.Address
    const accounts = try allocator.alloc(jsonrpc.types.Address, config.managed_accounts.len);
    for (config.managed_accounts, 0..) |addr, i| {
        accounts[i] = .{ .bytes = addr.bytes };
    }
    return eth_accounts.Result{
        .value = accounts,
    };
}
```

## Zig Style Compliance

1. **No local type aliases**: Use `primitives.Address` inline, not `const Address = primitives.Address`
2. **No stored allocators**: Pass allocator explicitly to methods that need it (e.g., `handleAccounts`)
3. **Fully qualified paths**: Use complete type paths throughout
