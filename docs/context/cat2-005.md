# Context: cat2-005 — Add voltaire jsonrpc module import to zevm build.zig

## Ticket Info
- **Ticket ID**: cat2-005
- **Category**: architecture
- **Goal**: Expose voltaire's `jsonrpc` module to zevm by adding it to `build.zig` imports, so that zevm handler code can use the existing JSON-RPC Params/Result type definitions.

---

## Problem Statement

`zevm/build.zig` imports 5 voltaire modules:
```zig
const primitives_mod = voltaire.module("primitives");
const state_manager_mod = voltaire.module("state-manager");
const blockchain_mod = voltaire.module("blockchain");
const crypto_mod = voltaire.module("crypto");
const precompiles_mod = voltaire.module("precompiles");
```

But it does **not** import the `jsonrpc` module. The `jsonrpc` module lives at:
- `../voltaire/packages/voltaire-zig/src/jsonrpc/`
- Root: `root.zig`

Without this import, zevm handlers cannot use the rich Params/Result types that voltaire already provides for all 65 Ethereum JSON-RPC methods.

---

## voltaire jsonrpc Module — What It Provides

### File Layout
```
../voltaire/packages/voltaire-zig/src/jsonrpc/
  root.zig          — module root, re-exports everything
  JsonRpc.zig       — root JsonRpcMethod union (engine | eth | debug)
  types.zig         — re-exports shared primitive types
  types/
    Address.zig     — 20-byte hex address with JSON serde
    Hash.zig        — 32-byte hex hash with JSON serde
    Quantity.zig    — hex-encoded unsigned integer
    BlockTag.zig    — "latest"/"earliest"/"pending" etc.
    BlockSpec.zig   — block number OR tag OR hash
  eth/
    methods.zig     — EthMethod union (40 methods)
    getBalance/     — eth_getBalance Params + Result
    call/           — eth_call Params + Result
    ... (38 more)
  debug/
    methods.zig     — DebugMethod union (5 methods)
    getBadBlocks/   — debug_getBadBlocks Params + Result
    ... (4 more)
  engine/
    methods.zig     — EngineMethod union (20 methods)
    forkchoiceUpdatedV1/ — engine_forkchoiceUpdatedV1 Params + Result
    ... (19 more)
```

### Public API (from `root.zig`)
```zig
pub const JsonRpc = @import("JsonRpc.zig");
pub const eth = @import("eth/methods.zig");
pub const debug = @import("debug/methods.zig");
pub const engine = @import("engine/methods.zig");
pub const types = @import("types.zig");
pub const JsonRpcMethod = JsonRpc.JsonRpcMethod;
```

### Key Types
- `JsonRpcMethod` — tagged union: `.engine`, `.eth`, `.debug`
- `eth.EthMethod` — tagged union of 40 eth_ methods, each with `.params` and `.result`
- `engine.EngineMethod` — tagged union of 20 engine_ methods
- `debug.DebugMethod` — tagged union of 5 debug_ methods
- `types.Address` — `[20]u8` with `0x`-prefixed hex JSON serde
- `types.Hash` — `[32]u8` with `0x`-prefixed hex JSON serde
- `types.Quantity` — wraps `std.json.Value`, represents hex uint
- `types.BlockTag` — wraps `std.json.Value` for `"latest"/"earliest"` etc.
- `types.BlockSpec` — union of block number | tag | hash

### Critical Finding: No External Dependencies
The `jsonrpc` module imports **only `std`** — no `primitives`, `crypto`, or other voltaire modules.
Its types (Address, Hash, etc.) are **independent** of `primitives.Address`, `primitives.Hash`.
This means adding it to voltaire's `build.zig` as a module requires **no dependency setup**.

## Required Changes

### Step 1: Add jsonrpc module to voltaire/build.zig

In `../voltaire/build.zig`, after the `precompiles_mod` declaration (around line 145), add:

```zig
// JSON-RPC types module — no external dependencies, pure std
const jsonrpc_mod = b.addModule("jsonrpc", .{
    .root_source_file = b.path("packages/voltaire-zig/src/jsonrpc/root.zig"),
    .target = target,
    .optimize = optimize,
});
```

No `addImport` calls needed — the module is self-contained (only uses `std`).

### Step 2: Add jsonrpc_mod to zevm/build.zig

**2a.** Fetch the module (after line 16 where other modules are fetched):
```zig
const jsonrpc_mod = voltaire.module("jsonrpc");
```

**2b.** Add to the `zevm` module imports:
```zig
const mod = b.addModule("zevm", .{
    .root_source_file = b.path("src/root.zig"),
    .target = target,
    .imports = &.{
        .{ .name = "primitives", .module = primitives_mod },
        .{ .name = "state-manager", .module = state_manager_mod },
        .{ .name = "blockchain", .module = blockchain_mod },
        .{ .name = "crypto", .module = crypto_mod },
        .{ .name = "precompiles", .module = precompiles_mod },
        .{ .name = "guillotine_mini", .module = guillotine_mini_mod },
```

## Usage After the Fix

Once wired in, zevm source files can use:
```zig
const jsonrpc = @import("jsonrpc");

// Access Params/Result for any method:
const params: jsonrpc.eth.getBalance.eth_getBalance.Params = ...;
const result: jsonrpc.eth.getBalance.eth_getBalance.Result = ...;

// Or via the method union:
const method: jsonrpc.JsonRpcMethod = ...;

// Shared types:
const addr: jsonrpc.types.Address = ...;
const qty: jsonrpc.types.Quantity = ...;
```

Note: `jsonrpc.types.Address` is NOT the same type as `primitives.Address` — they are separate types. Handler code that bridges between the two will need explicit conversion.

## Zig Style Notes
- Do NOT alias `const jsonrpc = @import("jsonrpc")` at file-top if only one type is used — use inline. But `@import("jsonrpc")` at top-of-file is the standard import pattern (the exception in CLAUDE.md).
- No stored allocators; `jsonParseFromValue` receives allocator as parameter.
- `jsonrpc.types.Address` and `primitives.Address` are distinct types — conversions must be explicit.
