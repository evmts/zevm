# Research Context: cat2-007 — JSON-RPC HTTP Server in main.zig with eth_* Dispatch

> Archival snapshot note: this document is non-normative and may contain historical Voltaire/EDR error-code examples.
> Normative ZEVM behavior is defined in `docs/specs/prd.md` and `docs/specs/json-rpc-contract.md` (with transport details in `docs/specs/internal/transport-and-error-semantics.md`).

## Ticket Summary

Implement an HTTP server in `main.zig` that:
1. Accepts POST requests on a configurable port
2. Parses JSON-RPC 2.0 request envelopes
3. Dispatches to handlers via method name
4. Serializes responses

Start with `-32601 Method not found` for all methods — this lets Hive rpc-compat tests run immediately and measures progress as handlers are added.

## JSON-RPC 2.0 Envelope Types to Add in Voltaire

**File to create:** `../voltaire/packages/voltaire-zig/src/jsonrpc/envelope.zig`

Based on the official JSON-RPC 2.0 spec, foundry anvil patterns, and EDR patterns:

```zig
const std = @import("std");

/// JSON-RPC 2.0 Request ID: string, integer, or null
pub const RequestId = union(enum) {
    integer: i64,
    string: []const u8,
    null_value,

    pub fn jsonStringify(self: RequestId, jws: *std.json.Stringify) !void {
        switch (self) {
            .integer => |n| try jws.write(n),
            .string => |s| try jws.write(s),
            .null_value => try jws.write(null),
        }
    }

    pub fn jsonParseFromValue(allocator: std.mem.Allocator, source: std.json.Value, options: std.json.ParseOptions) !RequestId {
        _ = options;
        return switch (source) {
            .integer => |n| .{ .integer = n },
            .float => |f| .{ .integer = @intFromFloat(f) },
            .string => |s| .{ .string = try allocator.dupe(u8, s) },
            .null => .null_value,
            else => error.UnexpectedToken,
        };
    }
};

/// JSON-RPC 2.0 Request envelope
pub const Request = struct {
    jsonrpc: []const u8,       // expected to equal "2.0"
    method: []const u8,
    params: ?std.json.Value,   // null, array, or object
    id: ?RequestId,            // null for notifications

    pub fn isNotification(self: Request) bool {
        return self.id == null;
    }

    pub fn isValid(self: Request) bool {
        return std.mem.eql(u8, self.jsonrpc, "2.0") and self.method.len > 0;
    }
};

/// JSON-RPC 2.0 Error object
pub const ErrorObject = struct {
    code: i32,
    message: []const u8,
    data: ?std.json.Value = null,
};

/// JSON-RPC 2.0 Response envelope
pub const Response = struct {
    jsonrpc: []const u8 = "2.0",
    id: ?RequestId,
    result: ?std.json.Value = null,
    @"error": ?ErrorObject = null,
};

/// Historical proposal example only.
/// ZEVM authoritative error codes are defined in docs/specs/json-rpc-contract.md section 5.
pub const ErrorCode = struct {
    pub const PARSE_ERROR:          i32 = -32700;
    pub const INVALID_REQUEST:      i32 = -32600;
    pub const METHOD_NOT_FOUND:     i32 = -32601;
    pub const INVALID_PARAMS:       i32 = -32602;
    pub const INTERNAL_ERROR:       i32 = -32603;
    // Ethereum-specific (per MetaMask/EIP-1474)
    pub const INVALID_INPUT:        i32 = -32000;
    pub const RESOURCE_NOT_FOUND:   i32 = -32001;
    pub const RESOURCE_UNAVAILABLE: i32 = -32002;
    pub const TRANSACTION_REJECTED: i32 = -32003;
    pub const METHOD_NOT_SUPPORTED: i32 = -32004;
    pub const LIMIT_EXCEEDED:       i32 = -32005;
};

/// Helper: create a method-not-found error response
pub fn methodNotFound(id: ?RequestId, method_name: []const u8, allocator: std.mem.Allocator) !Response {
    const msg = try std.fmt.allocPrint(allocator, "Method not found: {s}", .{method_name});
    return Response{
        .id = id,
        .@"error" = .{ .code = ErrorCode.METHOD_NOT_FOUND, .message = msg },
    };
}
```

## HTTP Server Implementation (std.http.Server)

**File to create:** `src/rpc_server.zig`

Zig 0.15 provides `std.http.Server` in stdlib. No external HTTP libraries needed.

### Key std.http.Server API (Zig 0.15)

```zig
var server = std.http.Server.init(allocator, .{ .reuse_address = true });
defer server.deinit();

const address = try std.net.Address.parseIp4("127.0.0.1", 8545);
try server.listen(address);

while (true) {
    var request = try server.accept(.{ .dynamic = allocator });
    defer request.deinit();
    try request.wait();  // read headers

    // Read body
    const body = try request.reader().readAllAlloc(allocator, max_body_size);
    defer allocator.free(body);

    // Send response
    try request.respond(response_bytes, .{
        .status = .ok,
        .extra_headers = &.{
            .{ .name = "content-type", .value = "application/json" },
            .{ .name = "access-control-allow-origin", .value = "*" },
        },
    });
}
```

```zig

if (request.method != .POST) {
```

### Method Dispatch via voltaire EthMethod.fromMethodName()

```zig
const method_str = request.method;

// Try eth namespace
if (std.mem.startsWith(u8, method_str, "eth_")) {
    const tag = @import("voltaire_jsonrpc").eth.EthMethod.fromMethodName(method_str) catch {
        return makeErrorResponse(request.id, ErrorCode.METHOD_NOT_FOUND, method_str);
    };
    return dispatchEthMethod(tag, request, state);

// Try debug namespace
if (std.mem.startsWith(u8, method_str, "debug_")) {
    const tag = @import("voltaire_jsonrpc").debug.DebugMethod.fromMethodName(method_str) catch {
        return makeErrorResponse(request.id, ErrorCode.METHOD_NOT_FOUND, method_str);
    };
    return dispatchDebugMethod(tag, request, state);
}

// Unknown namespace
return makeErrorResponse(request.id, ErrorCode.METHOD_NOT_FOUND, method_str);
```

## CLI Argument Parsing

**Standard Zig:** `std.process.ArgIterator` — no external library needed for dev node flags.

```zig
pub const Config = struct {
    port: u16 = 8545,
    host: []const u8 = "127.0.0.1",
    chain_id: u64 = 31337,  // hardhat/anvil default
    fork_url: ?[]const u8 = null,
    accounts: u32 = 10,
    block_time: ?u64 = null,  // null = automine
};

pub fn parseArgs(allocator: std.mem.Allocator) !Config {
    var it = try std.process.ArgIterator.initWithAllocator(allocator);
    defer it.deinit();
    _ = it.next(); // skip program name

    var config = Config{};
    while (it.next()) |arg| {
        if (std.mem.eql(u8, arg, "--port")) {
            const val = it.next() orelse return error.MissingValue;
            config.port = try std.fmt.parseInt(u16, val, 10);
        } else if (std.mem.eql(u8, arg, "--host")) {
            config.host = it.next() orelse return error.MissingValue;
        } else if (std.mem.eql(u8, arg, "--chain-id")) {
            const val = it.next() orelse return error.MissingValue;
            config.chain_id = try std.fmt.parseInt(u64, val, 10);
        } else if (std.mem.eql(u8, arg, "--fork-url")) {
            config.fork_url = it.next() orelse return error.MissingValue;
        }
    }
    return config;
}
```

## Design Decisions

### Use `std.http.Server` (stdlib)
- Sufficient for dev node — no need for httpz, zap, or other HTTP libs
- Single-threaded is fine for dev use; threading is future optimization
- Zig 0.15 `std.http.Server` API is stable

### Envelope types go in voltaire, not zevm
- voltaire-ts already has these types (`packages/voltaire-ts/src/jsonrpc/types.ts`)
- Snapshot proposal: Zig mirrors TypeScript so voltaire remains the source of truth
- Snapshot proposal: zevm imports `jsonrpc.envelope.Request`, `jsonrpc.envelope.Response`, etc.

### Method dispatch uses prefix routing + StaticStringMap
- `std.mem.startsWith(u8, method, "eth_")` → try `EthMethod.fromMethodName()`
- `std.mem.startsWith(u8, method, "debug_")` → try `DebugMethod.fromMethodName()`
- `std.mem.startsWith(u8, method, "engine_")` → try `EngineMethod.fromMethodName()`
- O(1) lookup within each namespace via `StaticStringMap`
- Unknown methods/namespaces → -32601

### Start with -32601 for all recognized methods
- All 199 hive tests will get proper error format immediately
- Measurable progress: hive pass count increases as handlers added
- Subsequent tickets (cat2-008+) add individual method implementations

### Sequential batch processing
- Matches EDR approach (state mutations are serialized)
- Simpler in Zig (no async, no threading)
- Foundry's concurrent batch is optimization for later

### No local type aliases (Zig style rule)
- Use `@import("jsonrpc").envelope.Request` inline
- Use `@import("jsonrpc").eth.EthMethod.fromMethodName()`
- Never: `const EthMethod = @import("jsonrpc").eth.EthMethod`
